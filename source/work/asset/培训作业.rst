+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 3、项目经理不能有效汇报工作，使他们自身在组织中的价值体现不足。如何指导项目经理进行工作汇报？                                                                                                                                                                                                                    |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 4、当项目经理过度下沉到具体任务和问题中去，导致工作量超载或放弃大局统筹。如何指导项目经理把握细节与大局的平衡？                                                                                                                                                                                                           |

+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 事件背景        | 1、B项目是公司近期重点项目；但不同于以往的研发项目，这是一个偏重交付的项目，且时间紧迫，需要在设计验证同时并行交付活动。先前梳理的研发流程不能充分适用。                                                                                                                                                                                      |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 2、C同学是部门重点培养的项目经理人选，具备较好的责任心，但缺乏对大型项目负责的成功经验。由于组织变化，C也是突然接受这个项目任务的。                                                                                                                                                                                                |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Challenge   | 在当时背景下，需要达成什么样的目标，面对什么样的阻碍或者问题，主人公在事件中面临的挑战或困惑，或者小白、新手一般会遭遇的难题及障碍是什么。                                                                                                                                                                                              |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 挑战、冲突       | 1、起初，项目计划不完备，仿佛总是会出现不在计划之内的新目标或新活动；在制定计划中，C可能将一些任务引发的新任务不断添加到计划中，这使计划越来越具体，但也越来越变得庞大和复杂，在现行的组织模式和工具背景下超越了他个人的管理负荷；渐渐地，更新与变更变得困难，项目面临计划失效的风险。                                                                                                                       |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 2、为了促成目标，C期望掌握所有问题和困难的真相，这使他不断深入到每一个任务细节中去，但还是有一些任务无法按计划完成，当C回头看这些的时候，会产生失望和焦虑。                                                                                                                                                                                    |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 3、C的项目报告会包含很多细节，但不能有效概括项目的总体绩效，而管理层对细节的兴趣有限，这就逐渐形成了一些隔膜。                                                                                                                                                                                                           |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Action      | 详略得当地叙述该事件发展的来龙去脉。                                                                                                                                                                                                                                                 |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 事件经历        | 1）大体按时间发展顺序，但切忌记流水账和面面俱到，针对1-2个关键点深挖                                                                                                                                                                                                                               |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 2）针对遇到的挑战和困难，是如何处理解决的，说了什么做了什么，是怎么盘算和分析的                                                                                                                                                                                                                           |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 3）包含当事人的必要提示，字体可以区别开来                                                                                                                                                                                                                                              |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 1、出于情急（前文说过C也是突然接受这个项目任务的），项目工作分解结构主要是由我（C的管理者）制定的，之后我花了较长的时间与之充分沟通这份计划的内容、结构以及各个任务的定义，我希望通过这种沟通使之充分理解计划的内涵。但这个沟通是不足以完成期望的，也为未来的管理工作预埋了隐患。                                                                                                                         |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 2、我将计划更新、管理的权限全部授予到C，并约定我会每周审视一次这些工作的成果。                                                                                                                                                                                                                           |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 3、从第3周开始，我意识到这些工作出现了偏差，到第4周就恶化得比较明显了，一些关键任务和风险的进展信息出现了一周以上的滞后。                                                                                                                                                                                                     |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 4、为了驱动C改善管理风格，我分析他的处境，认为主要有两点需要改进：（1）出于以往的习惯，对计划的重视程度不够；（2）过于沉浸到任务和困难的细节中去，往往化身成为某个任务的负责人去驱动一个任务的关闭，但这反倒可能使原来的任务负责人丧失了责任感。                                                                                                                                         |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 5、我与他沟通这些问题，促使他自己认识到这些问题。首先，对第（1）点，我跟他描述我们的完整的工作流：认识目标、制定大纲、分解结构、组织任务关系、识别风险、执行干预，这些环环相扣的活动缺一不可，而其中计划的重要性不言而喻。很快他就自己认可了这一点而不是通过我宣讲。之后，对第（2）点，我与他演绎了一个任务出现困难之后的处理方式，在演绎中，他也自己意识到可以采取更好的方式来平衡自己的工作。                                                                  |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 6、对于工作汇报的问题，我主张持续改进，由于每个项目经理都在业务实践中，抽出来总结归纳的角色就由我承担，我每个月对汇报模板进行一次优化，但每次改版都与每个项目经理充分沟通达成共识，也吸纳他们的建议。                                                                                                                                                                |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Result      | 概述事件最终处理后的结果，最终实现了什么样的一个结果，与目标偏离度大不大，有没有其他的外部影响，比如表彰和批评等。                                                                                                                                                                                                          |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 结果          | 1、项目经理的管理认识发生了一定程度改观，表现在：（1）对保持计划有效的重要性达成共识；（2）在经过共创梳理后，对任务结构的理解更为清晰，当新任务或新目标出现时也能妥当处理；（3）对任务、风险、对策这些概念以及概念的边界认识提高，这样就避免了把不同类型的对象混淆在一起，从而保持了任务结构的稳定。                                                                                                               |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 2、项目管理水平有一些改善但并没有取得立竿见影的成绩。改善主要在于：（1）项目计划管理最终回到了有效可控的正轨上；（2）汇报模式有一定改观；（3）项目经理逐渐主动意识到责任分解的重要性，各领域组长的责任意识也有一定改观，例如测试中的非技术问题，测试组长也更主动解决；现场负责人为保证施工结果，开始主动与研发沟通技术细节（这些问题以往可能会经过项目经理中转）；（4）项目经理将更多的精力投入到风险管理中，先前疏漏的风险跟踪与干预水平有提高；（5）采取一些量化方法将项目绩效可视化，这在多项目管理中可能会更有参考意义。  |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 3、不足在于：（1）在工作分解这件工作中，没有使员工通过实践获得成功经验；（2）由于“（1）”的原因，导致员工在跟踪计划中对一些任务的定义、关系理解出现了偏差，纠正这些偏差的成本甚至抵消了我原来代为编制工作分解试图节约的；（3）通过一个范式实践可能不足以解决的问题：项目经理的内在驱动力、坚定达成目标不动摇的意志、追求结果导向的意识，而宣讲、沟通也没有使这些显著提升，可能还需要通过更多实践经历来提高。                                                          |

+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| 总结分析        | 1、对他人：（1）积极引导还是有意义的，并非听不进去；（2）实践是最好的老师。                                                                                                                                                                                                                            |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 2、对自己：（1）还是不自觉地选择了对自己最容易的处理方法，这是一种怠惰。例如，在制定计划阶段，选择自己整理编再沟通交流的方式制其实是一种怠惰，因为预判了“在沟通取得共识的条件下共创”可能是更困难的路径。（2）感到自己对他人的要求不够高，在对方工作没有达到预期结果时会选择理解包容，或许会自己修改完善，也或许会等待下次机会纠正，不愿意当即鞭策或督促改善，这样的本意是为了重视工作效率，不在一个细节上过度纠缠，但也可能因此荫蔽了一些员工不够认真严谨的作风，我要加强“指导工作”课程中对“批评”的学习实践。        |
+-------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
|             | 3、对方法：当我在提供一种项目管理实践范式的时候，尽管我自己很推崇，但必须承让其本身也存在不完美，受限于全员对项目管理体系的理解配合程度、工具水平、项目经理 个人素养，实践只达到了范式期望的70%水平。我从中总结意识到的：（1）相对于追求精益来说，方法更重要要结合实际，利于操作是精益的前提，反之则不然；（2）方法要与工具结合，保持凝练，这既有利于操作的作用，也有保证细节的作用，工具设计、开发是对项目管理体系建设不可或缺的支撑，这方面我们尽管做了一些，但还要不断提高。                        |
